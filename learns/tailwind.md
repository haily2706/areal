# Tailwind CSS & CVA Learning Plan

## Overview

This plan takes you from zero to confident with Tailwind CSS and Class Variance Authority (CVA). Estimated total time: 2-3 weeks at 1-2 hours/day.

---

## Phase 1: Tailwind CSS Fundamentals (Week 1)

### Module 1.1: Core Concepts (Day 1-2)
- [ ] Understand utility-first CSS philosophy
- [ ] Learn the class naming convention
- [ ] Practice: spacing, sizing, colors

### Module 1.2: Layout & Positioning (Day 3-4)
- [ ] Flexbox utilities
- [ ] Grid utilities
- [ ] Positioning (absolute, relative, fixed)
- [ ] Practice: Build a navbar and card layout

### Module 1.3: Typography & Colors (Day 5)
- [ ] Font sizes, weights, families
- [ ] Text alignment and decoration
- [ ] Color system (bg-, text-, border-)
- [ ] Practice: Style a blog post layout

### Module 1.4: Responsive Design (Day 6)
- [ ] Breakpoint prefixes (sm:, md:, lg:, xl:, 2xl:)
- [ ] Mobile-first approach
- [ ] Practice: Make previous layouts responsive

### Module 1.5: States & Interactivity (Day 7)
- [ ] Hover, focus, active states
- [ ] Group and peer modifiers
- [ ] Transitions and animations
- [ ] Practice: Interactive button and form

---

## Phase 2: Tailwind Intermediate (Week 2, Days 1-3)

### Module 2.1: Customization
- [ ] tailwind.config.js structure
- [ ] Extending the theme
- [ ] Custom colors and spacing
- [ ] Adding custom fonts

### Module 2.2: Advanced Patterns
- [ ] Arbitrary values [color:#123456]
- [ ] @apply directive (and when NOT to use it)
- [ ] Dark mode (class and media strategies)

### Module 2.3: Real Components
- [ ] Build: Button variations
- [ ] Build: Form inputs
- [ ] Build: Card component
- [ ] Build: Modal/Dialog

---

## Phase 3: CVA Fundamentals (Week 2, Days 4-7)

### Module 3.1: Why CVA? (Day 4)
- [ ] Problems with conditional classes
- [ ] CVA's solution: structured variants
- [ ] Installation and setup

### Module 3.2: Basic CVA Patterns (Day 5)
- [ ] Base styles
- [ ] Defining variants
- [ ] Default variants
- [ ] TypeScript integration with VariantProps

### Module 3.3: Advanced CVA (Day 6)
- [ ] Compound variants
- [ ] Composing styles
- [ ] Integration with clsx/tailwind-merge

### Module 3.4: CVA in Practice (Day 7)
- [ ] Build: Complete Button component
- [ ] Build: Badge/Tag component
- [ ] Build: Alert component
- [ ] Build: Input component

---

## Phase 4: Integration & Projects (Week 3)

### Module 4.1: The cn() Pattern
- [ ] Understanding clsx
- [ ] Understanding tailwind-merge
- [ ] Creating the cn() utility
- [ ] When and why to use it

### Module 4.2: Component Library Patterns
- [ ] Folder structure
- [ ] Props patterns
- [ ] Forwarding refs
- [ ] Composition patterns

### Module 4.3: Final Project
- [ ] Build a mini component library with:
    - Button (with variants)
    - Input (with states)
    - Card (with variants)
    - Badge
    - Alert
